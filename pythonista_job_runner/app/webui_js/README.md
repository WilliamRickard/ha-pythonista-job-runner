# Web UI JavaScript parts

The Web UI JavaScript is split into multiple files for readability and review.

Do not edit `pythonista_job_runner/app/webui.js` directly. It is generated by `pythonista_job_runner/app/webui_build.py` from the parts in this folder, and then inlined into the bundled `webui.html`.

## Assembly order

These files are concatenated in an explicit order defined by `WEBUI_JS_PARTS` in `pythonista_job_runner/app/webui_build.py`.
If you add, remove, or rename a part, update that list too. The build fails if a file is missing or if an unexpected `*.js` file exists in this folder.

## Rules

- Do not add `VERSION:` headers in part files. The bundle header is generated at the top of `webui.js`.
- Avoid root-relative requests like `fetch('/api')`. Home Assistant Ingress runs add-ons under a path prefix; the build fails and reports file and line number.
- Keep each file focused (rendering, event wiring, polling, detail view) so diffs stay small.

## Rebuild and check

```bash
python pythonista_job_runner/app/webui_build.py
python pythonista_job_runner/app/webui_build.py --check
```
