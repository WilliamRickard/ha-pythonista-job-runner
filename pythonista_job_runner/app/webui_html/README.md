# Web UI HTML parts

## Version headers

These READMEs are intentionally versionless. The Web UI version is controlled by `WEBUI_VERSION` in `pythonista_job_runner/app/webui_build.py`.
Do not add a line like `<!-- Version: ... -->` here; the build fails if it appears.

This folder contains the source HTML *body* split into parts for maintainability.

The runtime UI is still served as a single bundled `webui.html` (to avoid Home Assistant Ingress path-prefix issues). `webui.html` is generated by `pythonista_job_runner/app/webui_build.py` from:

- `webui_src.html` (thin wrapper with placeholders)
- `webui_html/*.html` (these body parts)
- `webui_css/*.css` (CSS parts, assembled into generated `webui.css`)
- `webui_js/*.js` (JS parts, assembled into generated `webui.js`)

## Assembly order

The build uses an explicit ordered list (`WEBUI_HTML_PARTS`) in `pythonista_job_runner/app/webui_build.py`.
If you add, remove, or rename a part, update that list too.

## Rules

- Do not include document-level tags (`<!doctype>`, `<html>`, `<head>`, `<body>`).
- Do not include `<script>` or `<style>` blocks. Use `webui_js/*.js` and `webui_css/*.css`.
- Keep HTML `id` attributes unique across all parts. The build fails on duplicates and reports the files involved.
- Avoid root-relative URLs (for example `href="/..."` or `src="/..."`). Home Assistant Ingress runs under a path prefix; the build fails and reports file and line number.

## Rebuild and check

From the repo root:

```bash
python pythonista_job_runner/app/webui_build.py
python pythonista_job_runner/app/webui_build.py --check
```

CI also enforces this via the test suite (see `pythonista_job_runner/tests/test_webui_bundle.py`).
